1.1 DELETE INDIVIDUAL SETS
    Deskripsi: 
    Memungkinkan pengguna untuk menghapus satu set spesifik di dalam latihan tanpa harus menghapus seluruh latihan atau mereset ulang input. Tombol hapus akan muncul saat pengguna melakukan swipe pada set row atau melalui menu opsi.
    
    Teknis:
    Saat ini `WorkoutProvider` hanya memiliki method `removeExercise` tetapi tidak ada method `removeSet`. Di `workout_logger_screen.dart`, callback `onDeleteSet` sudah ada tetapi hanya menampilkan snackbar dummy. Kita perlu mengimplementasikan logika manipulasi List di provider untuk menghapus item pada index tertentu dari array `sets` di dalam `ActiveExercise`.
    
    Kesulitan: ‚≠ê Mudah
    Kebutuhan: Tambah method `deleteSet(int exIndex, int setIndex)` di provider.
    Butuh Bantuan Manual: ‚ùå Tidak

1.2 SET TYPES SELECTION (Warmup/Failure)
    Deskripsi: 
    Menambahkan kemampuan untuk menandai set sebagai 'Warmup', 'Normal', 'Failure', atau 'Dropset'. Ini penting untuk analitik yang akurat karena set pemanasan tidak seharusnya dihitung dalam kalkulasi PR (Personal Record).
    
    Teknis:
    Model `SetType` (enum) dan properti `type` di `WorkoutSet` sudah ada di `models.dart`. UI chip selector juga sudah dibuat di `workout_logger_screen.dart` tetapi belum terhubung ke logic update. Masalahnya adalah nilai default yang mungkin salah index dan UI yang belum mentrigger `provider.updateSet` dengan parameter tipe set.
    
    Kesulitan: ‚≠ê Mudah
    Kebutuhan: Hubungkan UI chip ke `provider.updateSet(..., type: newType)`.
    Butuh Bantuan Manual: ‚ùå Tidak

1.3 RPE INPUT
    Deskripsi: 
    Menambahkan kolom input untuk RPE (Rate of Perceived Exertion) skala 1-10 untuk setiap set. Ini membantu melacak intensitas latihan subjektif di luar beban dan repetisi.
    
    Teknis:
    Model `WorkoutSet` sudah memiliki field `rpe`, tetapi belum ada UI input field di `SetRow` widget. Kita perlu menambahkan `GymInput` kecil di sebelah kolom Reps atau di menu visual terpisah baris set, dan memastikan validasi input hanya menerima angka 1-10 (atau 0.5 step).
    
    Kesulitan: ‚≠ê Mudah
    Kebutuhan: Tambah `GymInput` field untuk RPE dan logic update.
    Butuh Bantuan Manual: ‚ùå Tidak

1.4 CONFIGURABLE REST TIMER
    Deskripsi: 
    Membuat pengaturan durasi istirahat di profil benar-benar berfungsi. Saat ini pengguna bisa mengubah durasi default di pengaturan, tapi timer saat workout masih menggunakan nilai hardcoded atau nilai lama.
    
    Teknis:
    `SettingsProvider` sudah menyimpan `defaultRestSeconds`, tetapi `WorkoutProvider` tidak membacanya saat menginisialisasi timer istirahat. Diperlukan sinkronisasi agar saat tombol "Rest" ditekan, durasi diambil dari `SettingsProvider` atau override lokal per exercise.
    
    Kesulitan: ‚≠ê Mudah

    Butuh Bantuan Manual: ‚ùå Tidak

4.1 CLOUD BACKUP (Firebase/Supabase)
    Deskripsi: 
    Sinkronisasi data agar aman jika HP hilang atau ganti HP.
    
    Teknis:
    Aplikasi perlu koneksi ke Backend-as-a-Service (BaaS). SDK Firebase/Supabase perlu file konfigurasi unik yang berisi API Keys project Anda.
    
    ‚ö†Ô∏è MENGAPA BUTUH BANTUAN MANUAL:
    AI tidak bisa login ke Google Account/Gmail Anda untuk membuat project di Firebase Console. AI tidak bisa generate API Key billing untuk Anda.
    
    Langkah Anda:
    1. Pergi ke https://console.firebase.google.com
    2. Buat project baru "LiftNode"
    3. Enable Firestore Database & Auth
    4. Download `google-services.json` dan letakkan di folder android/app
    5. Beritahu AI file sudah ada, baru AI bisa setting Gradle config-nya.

4.2 PROGRESS PHOTOS
    Deskripsi: 
    Mengambil foto dari kamera atau galeri untuk tracking fisik.
    
    Teknis:
    Integrasi plugin `image_picker` dan akses storage file system lokal.
    
    ‚ö†Ô∏è MENGAPA BUTUH BANTUAN MANUAL:
    Emulator/Simulator kamera sering error atau blank screen. Testing permission (Minta akses kamera/galeri, popup "Allow/Deny") sangat sulit didebug tanpa interaksi fisik di layar device. Anda harus run di HP asli untuk memastikan tidak crash.

4.3 EXERCISE IMAGES/ANIMATIONS
    Deskripsi: 
    Visualisasi gerakan latihan (GIF/Gambar statis) di library agar user tahu cara melakukannya.
    
    Teknis:
    Aplikasi butuh file aset gambar (misal: bench_press.gif, squat.png). Total bisa ratusan file.
    
    ‚ö†Ô∏è MENGAPA BUTUH BANTUAN MANUAL:
    AI bisa generate kode path gambar, tapi TIDAK BISA mendownload/membuat 100+ file gambar latihan yang akurat secara anatomis dan memasukkannya ke folder `assets/`.
    Opsi Anda:
    A. Cari dataset gambar open source (wger.de), download manual satu folder, zip, upload ke project.
    B. Minta AI generate satu-satu (lama & boros kuota).
    C. Beli asset pack icon fitness.
